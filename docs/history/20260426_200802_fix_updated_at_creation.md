# 🏁 2026-04-26 20:08:02 - 행 생성 시 updated_at 누락 수정

신규 행 생성 시 타임스탬프가 기록되지 않아 정렬 시 데이터가 예기치 않은 위치에 노출되는 문제를 해결하였습니다.

## 🛠️ 주요 변경 사항

### 1. 신규 행 생성 로직 보강 (Server/CRUD)
`create_empty_rows_batch` 함수에서 `models.DataRow` 객체를 생성할 때 `updated_at` 필드에 `func.now()`를 명시적으로 할당하도록 수정하였습니다.

- **기존**: `updated_at`이 할당되지 않아 DB에 `NULL`로 저장됨.
- **변경**: 생성 즉시 현재 시간이 타임스탬프로 기록됨.

### 2. 성능 중심의 정렬 유지
`NULLS LAST`와 같은 추가적인 정렬 옵션은 인덱스 효율 저하 우려로 인해 적용하지 않았으며, 생성 시점의 데이터 정합성을 확보하는 근본적인 방식으로 문제를 해결하였습니다.

## 📈 아키텍처 영향 보고
- **데이터 무결성**: 앞으로 생성되는 모든 행은 유효한 `updated_at` 값을 가지게 되어 정렬 일관성이 보장됩니다.
- **성능 유지**: DB 레벨의 추가 연산 없이 애플리케이션 레벨에서 타임스탬프를 주입함으로써 기존의 고속 인덱스 스캔 성능을 그대로 유지합니다.
