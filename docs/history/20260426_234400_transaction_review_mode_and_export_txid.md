# 트랜잭션 리뷰 모드 도입 및 익스포트 파라미터 확장

## 1. 개요
특정 인제션(트랜잭션) 결과를 테이블 뷰에서 한정하여 검수할 수 있는 **'트랜잭션 리뷰 모드'**를 도입하고, 대량 데이터 추출(CSV Export) 시에도 동일한 필터를 적용할 수 있도록 기능을 확장하였습니다.

## 2. 주요 변경 사항

### A. 서버 API 필터링 엔진 확장 (`server/main.py`)
`get_table_data`, `get_target_row_ids`, `export_table_csv` 등 주요 조회 API에 `transaction_id` 파라미터를 추가하였습니다. SQLAlchemy의 서브쿼리 조인을 활용하여 `AuditLog` 테이블에 해당 트랜잭션 기록이 있는 `DataRow`들만 인덱스 스캔으로 정밀 필터링합니다.

### B. 클라이언트 데이터 모델 및 필터 제어 (`client/models/table_model.py`)
`ApiLazyTableModel` 내부에 `_tx_filter` 상태를 관리하고, 무한 스크롤 및 전체 행 개수 동기화 시 해당 필터가 서버로 투명하게 전파되도록 페칭 파이프라인을 수정하였습니다.

### C. 히스토리 기반 컨텍스트 메뉴 도입 (`client/ui/panel_history.py`)
히스토리 패널의 요약 항목(트랜잭션 그룹) 클릭 시 **'🔍 이 결과만 테이블에서 보기'** 메뉴를 제공합니다. 이를 통해 사용자는 수많은 데이터 중 특정 인제션으로 인해 변경된 행들로만 뷰포트를 즉시 가둘 수 있습니다.

### D. 필터 상태 시각화 및 해제 (`client/ui/panel_filter.py`)
트랜잭션 필터가 활성화되었을 때, 상단 툴바에 필터 상태를 알리는 인디케이터와 **'❌ 필터 해제'** 버튼을 배치하여 사용자가 현재 어떤 컨텍스트에서 작업 중인지 명확히 인지하고 즉시 원복할 수 있도록 UX를 개선하였습니다.

### E. 익스포트 추적성 강화 (`client/main.py`)
CSV 추출 시 현재 활성화된 트랜잭션 필터가 자동으로 적용되도록 수정하였습니다. 이를 통해 10만 건의 인제션 결과 중 문제가 발생한 특정 트랜잭션 범위만 따로 CSV로 저장하여 검토할 수 있습니다.
