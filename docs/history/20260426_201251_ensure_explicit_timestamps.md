# 🏁 2026-04-26 20:12:51 - CRUD 데이터 변경 시 updated_at 명시적 갱신 보강

데이터 수정 경로에 관계없이 항상 최신의 타임스탬프를 보장하기 위해 CRUD 코어 로직을 보강하였습니다.

## 🛠️ 주요 변경 사항

### 1. 전방위적 타임스탬프 갱신 (Server/CRUD)
데이터가 실질적으로 변경되는 모든 핵심 함수에 `row.updated_at = func.now()`를 명시적으로 삽입하였습니다.

- **대상 함수**:
    - `apply_row_update_internal`: 데이터 업서트 및 수정 시
    - `delete_cell_source`: 특정 데이터 소스 삭제 시
    - `set_cell_manual_priority`: 수동 우선순위(Pin) 설정 시

### 2. 정합성 및 성능 보장
- SQLAlchemy의 `onupdate` 자동 갱신 정책에만 의존하지 않고 명시적으로 값을 주입함으로써, 복잡한 JSONB 내부 수정 시에도 정렬 순서가 흐트러지지 않도록 보장합니다.

## 📈 아키텍처 영향 보고
- **정렬 신뢰도**: 사용자가 정렬 기준을 변경하거나 데이터를 수정했을 때, 즉시 의도한 정렬 위치(최신순 등)에 반영되어 사용자 경험이 향상됩니다.
- **데이터 추적**: 모든 유효한 데이터 변경 사항이 `updated_at`에 정확히 기록됩니다.
