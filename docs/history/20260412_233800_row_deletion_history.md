# 프로젝트 이력: 행 삭제(Row Delete) 실시간 히스토리 연동 (2026-04-12)

## 1. 개요
데이터의 삭제 내역이 히스토리 패널에 기록되지 않아 사용자가 삭제 여부를 실시간으로 파악하기 어려웠던 결함을 해결함.

## 2. 상세 작업 내용

### 2.1 삭제 전용 시그널 신설 (`ApiLazyTableModel`)
- WebSocket의 `row_delete` 이벤트를 수신하여 외부로 전파할 수 있는 `row_deleted_ws` 시그널을 모델에 추가함.
- 행 삭제 성공 여부와 관계없이 브로드캐스트 메시지 수신 시 해당 시그널을 방출하여 모든 클라이언트가 삭제 이벤트를 인지하도록 함.

### 2.2 히스토리 패널 로그 생성 로직 추가 (`HistoryDockPanel`)
- `_log_row_deleted` 메서드를 구현하여 삭제된 행의 `row_id`와 테이블 정보를 히스토리에 기록함.
- **UI 스타일**: 🗑️ [삭제] 접두어와 빨간색(#f38ba8) 전경색을 적용하여 파괴적인 변경 사항임을 시각적으로 강조함.

## 3. 최종 결과
- 행 삭제 시 모델에서 데이터가 제거됨과 동시에, 우측 히스토리 패널에 해당 삭제 내역이 즉시 기록되는 것을 확인함.
