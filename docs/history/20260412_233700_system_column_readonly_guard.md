# 프로젝트 이력: 시스템 컬럼 읽기 전용(Read-Only) 보안 정책 구현 (2026-04-12)

## 1. 개요
데이터의 무결성과 신뢰성을 보장하기 위해, 시스템에 의해 자동 관리되는 컬럼(`created_at`, `updated_at`, `row_id`, `id`, `updated_by`)을 사용자가 임의로 수정하지 못하도록 봉쇄함.

## 2. 상세 작업 내용

### 2.1 클라이언트 모델 보호 (`ApiLazyTableModel`)
- `flags()` 메서드 수정: 시스템 컬럼에 대해 `ItemIsEditable` 플래그를 제외하여 UI 상에서 더블 클릭 및 편집이 불가능하도록 조치함.
- `setData` 및 `bulkUpdateData` 보호: API 요청 생성 전 시스템 컬럼 여부를 체크하여 수동 입력이나 붙여넣기(Paste) 시에도 해당 컬럼이 업데이트 대상에 포함되지 않도록 차단함.

### 2.2 서버 CRUD 레이어 보호 (`server/database/crud.py`)
- `update_cell` 수정: 특정 컬럼 이름이 시스템 컬럼일 경우 처리를 중단하고 `None`을 반환하도록 차단 로직 추가.
- `upsert_row` 수정: 외부 파서나 API에서 전달하는 `updates` 딕셔너리에서 시스템 컬럼 항목을 자동으로 필터링(Drop)하여 서버의 기록을 보호함.

## 3. 최종 결과
- UI 및 API 모든 경로에서 시스템 컬럼의 임의 수정이 불가능해졌으며, 오직 시스템 내부 로직에 의해서만 해당 데이터가 갱신됨을 확인함.
