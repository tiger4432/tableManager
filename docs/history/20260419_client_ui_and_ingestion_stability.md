# [Version / Fix] 2026.04.19 - 클라이언트 테마 강제화 및 인제션/웹소켓 무결성 패치

## 📝 개요
이 패치는 프론트엔드 UI의 환경 독립성 확보, 대기업/망분리 환경에서의 원활한 파싱 데이터 업로드 보장, 그리고 대규모 실시간 데이터 인입 시 발생하는 클라이언트 논리적 크래시를 방어하기 위해 적용되었습니다.

## 🛠️ 주요 변경 사항

### 1. `client/main.py`: Catppuccin 다크 테마 강제 렌더링
*   **배경**: 기존 Fusion 스타일은 Windows OS의 '밝게(Light)' 또는 '어둡게(Dark)' 설정에 종속되어, 어두운 계열로 디자인된 상태바/툴바 프레임과 네이티브 위젯(메시지 박스, 표 등) 간의 극단적인 명암비 불균형(눈부심 현상)을 유발했습니다.
*   **해결**: `QApplication` 초기화 직후 독자적인 **`QPalette`**를 생성하여 `Window`, `Base`, `Text`, `Button` 등 모든 렌더링 계층에 Catppuccin Mocha 팔레트 컬러를 하드코딩 주입했습니다. OS 환경 변수를 완전히 무시하고 일관된 테마를 유지합니다.

### 2. `server/parsers/directory_watcher.py`: 사내 Proxy 403 인증 에러 우회 (NO_PROXY)
*   **배경**: 로컬 폴더를 스캐닝하여 파싱 된 데이터를 서버 자체 API(`127.0.0.1:8000`)로 업서트(Push)하는 과정에서, 사내 보안 파이어월(Proxy)이 로컬 호스트 통신까지 인터셉트하여 `403 Forbidden` 에러를 응답, 인제션이 실패하는 이슈가 발생했습니다.
*   **해결**: 파이썬 파일의 글로벌 컨텍스트 단에서 `os.environ["NO_PROXY"] = "127.0.0.1,localhost,::1"` 변수를 주입하고, 빈 `urllib.request.ProxyHandler({})`를 내장 `urllib` 글로벌 오프너로 `install_opener`하여 모든 시스템 프록시를 완전히 패스하도록 격리했습니다.

### 3. `client/models/table_model.py`: 대규모 웹소켓 인제션 `IndexError` 크래시 방어
*   **배경**: 파일 업로드로 인해 수천 개의 `batch_row_upsert` 이벤트가 동시다발적으로 쏟아질 때, 백그라운드 클라이언트 스레드가 테이블 내 행 인덱스를 놓쳐 파이썬 프로세스 자체가 강제 종료되었습니다.
*   **원인**: 기존 행에 데이터를 병합하기 위해 리스트에서 `pop(idx)`을 수행한 "찰나의 순간"에 인덱스 지도를 재생성(`_build_row_id_map()`)해버려, 그 이후로 들어오는 대량의 행들의 주소 번지수(Index)가 모두 1씩 어긋나는(Off-by-one) 현상이 있었습니다.
*   **해결**: 대상 행을 수정한 후, 반드시 리스트 원위치에 `insert(idx)` 한 **그 직후**에만 인덱스 맵을 재생성하도록 호출 시점을 이동시켰습니다. 극단적인 데이터 스파이크 환경에서도 메모리 인덱스의 밀림이나 유실을 완벽하게 차단합니다.
