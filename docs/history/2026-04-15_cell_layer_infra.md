# 🗓️ 2026-04-15 마일스톤: 셀 레이어 제어 및 실시간 데이터 인제션 고도화

## 🎯 주요 목표 달성
오늘 세션에서는 데이터의 정밀 제어(Granular Control)와 사용자 편의성(UX)을 대폭 강화하는 핵심 인프라 작업을 마무리했습니다.

---

## 🚀 1. 셀 데이터 레이어링 및 원천 제어 (Cell Layer Management)
단일 값 저장 방식에서 벗어나, 데이터의 **'유래(Lineage)'**를 추적하고 사용자가 직접 선택할 수 있는 시스템을 구축했습니다.

- **데이터 중첩 모델**: 하나의 셀에 매뉴얼 수정값(`user`)과 여러 자동 파서(`parser_a`, `parser_b` 등)의 데이터가 공존하며, 시스템은 이를 레이어 형태로 저장합니다.
- **수동 고정(Pinning)**: 특정 소스의 데이터가 가장 정확하다고 판단될 경우, 해당 소스를 'Pin'하여 우선순위 엔진의 자동 결정을 오버라이드할 수 있습니다.
- **레이어 삭제**: 원치 않는 데이터 원천(잘못된 파서 결과 등)을 셀 단위에서 영구 삭제할 수 있는 기능을 추가했습니다.
- **데이터 계보(Lineage)**: `AuditLog`와 연동하여 셀의 모든 변경 이력을 시각화하고, 누가(OS User) 어떤 액션(`delete_source`, `set_priority`)을 취했는지 추적합니다.

## 📥 2. 범용 파일 업로드 및 자동화 파이프라인
클라이언트와 서버 간의 데이터 유입 장벽을 완전히 제거했습니다.

- **드래그 앤 드롭 업로드**: 로컬 파일을 테이블 탭으로 던지면 즉시 서버의 `ingestion_workspace`로 전송됩니다.
- **실시간 인제션 자동화**: 서버는 파일 유입을 즉시 감지하여 백그라운드에서 파서를 실행하고, 결과를 DB에 적재합니다.
- **엔드-투-엔드 동기화**: DB 적재 결과는 WebSocket을 통해 현재 탭을 열고 있는 모든 클라이언트 화면에 실시간으로 반영(노란색 하이라이트 등)됩니다.

## 👤 3. 사용자 추적 및 UX 안정화
협업 환경에서의 정합성과 UI 신뢰도를 높였습니다.

- **OS Identity Tracking**: 각 수동 수정 시 실행 환경의 OS 사용자 이름(예: `kk980`)을 자동으로 채득하여 모든 히스토리에 남깁니다.
- **UI/UX 폴리싱**:
    - `CellSourceManageDialog`: Catppuccin 스타일의 세련된 UI와 행 단위 클릭 모드 지원.
    - `History Panel`: `[User]` 태그 방식의 로그 포맷 개선으로 가독성 극대화.
    - **버그 해결**: `QColor` 임포트 오류 및 비동기 환경의 정적 데이터 정합성 강화.
