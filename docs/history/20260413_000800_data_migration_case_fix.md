# 프로젝트 이력: 데이터 무결성 복구 및 필드명 통합 마이그레이션 (2026-04-13)

## 1. 개요
커스텀 파서 도입 초기, 필드명 대소문자 매칭 미비로 인해 `production_plan` 테이블에 중복 필드(`PLAN_ID` vs `plan_id`)가 생성된 문제를 해결하고 데이터를 정규화함.

## 2. 상세 작업 내용

### 2.1 마이그레이션 스크립트 (`fix_case_data.py`) 구현
- **지능형 병합**: 각 행을 전수 조사하여 비정규 대문자 필드의 `sources`(원천 데이터)를 정규 소문자 필드로 안전하게 이동(Merge)함.
- **최종 값 갱신**: 데이터 병합 후 시스템의 우선순위 산정 로직(`compute_priority_value`)을 재실행하여 최종 표출 값을 최신화함.
- **데이터 정리**: 병합이 완료된 불필요한 대문자 필드를 삭제하여 데이터베이스 용량 최적화 및 가시성을 확보함.

### 2.2 자동 교정 방어 체계 연동
- 마이그레이션과 병행하여 `DirectoryWatcher`에도 지능형 정규화 로직을 탑재함으로써, 향후 발생하는 모든 커스텀 인제션 데이터가 자동으로 정규 필드에 매핑되도록 보장함.

## 3. 최종 결과
- `production_plan` 내 오염된 300여 개의 행 중 문제가 발견된 데이터들을 전량 복구 완료함.
- 이제 UI에서 대소문자 구분 없이 모든 데이터가 정규 컬럼에 일관되게 표시됨을 확인함.
