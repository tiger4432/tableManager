# 프로젝트 이력: 비가시 부상 행(Float-to-top) 히스토리 연동 및 네비게이션 강화 (2026-04-12)

## 1. 개요
로딩되지 않은 하위 데이터가 실시간으로 최상단으로 부상(Float-to-top)할 때, 해당 내역이 히스토리 패널에 기록되지 않거나 기록된 로그를 클릭해도 해당 행으로 이동하지 않는 네비게이션 결함을 해결함.

## 2. 상세 작업 내용

### 2.1 시그널 방출 흐름 개선 (`ApiLazyTableModel`)
- **문제**: 로컬 버퍼에 행이 없는 경우 모델 페칭 과정에서 조기 반환(`return early`)이 일어나 `ws_data_changed` 시그널이 방출되지 않음.
- **해결**: 시그널 방출 로직을 모델 업데이트 루프 상단으로 격상하여, 로컬 반영 여부와 관계없이 모든 원격 이벤트를 히스토리 패널에 즉시 전파하도록 수정함.

### 2.2 동적 행 추적(Dynamic Row Tracking) 구현 (`HistoryDockPanel`)
- **문제**: 로그 생성 시점에 행 위치를 알 수 없으면 클릭 시 `scrollTo` 기능이 작동하지 않음.
- **해결**: `_on_item_clicked` 시점에 로그 텍스트에서 `row_id`를 정규식으로 파싱하고, 모델의 `row_id_map`을 통해 실시간 위치를 재탐색하는 로직을 추가함.
- **결과**: 로그 생성 후 데이터가 최상단으로 이동하거나 뒤늦게 페칭되더라도, 클릭 시 정확한 행 위치를 찾아 이동함.

## 3. 최종 결과
- 보이지 않던 수천 번째 데이터가 수정되는 즉시 히스토리 패널에 로그가 남으며, 해당 로그 클릭 시 최상단으로 부상한 행으로 정확히 포커싱 및 스크롤되는 것을 확인함.
