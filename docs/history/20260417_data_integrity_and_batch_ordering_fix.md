# 데이터 정합성 및 배치 처리 순서 보정 작업 내역 (2026-04-17)

## 1. 개요
최근 통합 API 적용 이후 발생한 실시간 데이터 동기화 과정에서의 UI 결함(컬럼 증발, 행 중복, 순서 역전)을 정밀 분석하고 해결하였습니다.

## 2. 주요 수정 내용

### 🛰️ 서버 측 (Server-side)
- **컬럼 스키마 조회 보완**: `display_columns` 설정 여부와 관계없이 `created_at`, `updated_at`이 항상 포함되도록 `/schema` 엔드포인트 수정.
- **브로드캐스트 페이로드 강화**: 배치 업데이트(`batch_row_upsert`) 시 모든 행에 시간 메타데이터가 포함되도록 `inject_system_columns` 호출 추가.
- **데이터 중복 제거**: 단일 배치 내 동일 행에 대한 여러 업데이트가 있을 경우, 브로드캐스트 리스트에서 중복을 제거하여 신호 노이즈 최소화.

### 🖥️ 클라이언트 측 (Client-side)
- **데이터 병합(Merge) 아키텍처 도입**: WebSocket 수신 시 기존 행을 제거하지 않고, 기존 데이터와 새 데이터를 병합하여 `created_at` 등 기존 메타데이터의 영속성 보장.
- **중복 삽입 로직 제거**: 루프 내 중복된 `insert(0)` 호출을 제거하여 데이터 변경 시 행이 2개씩 생기던 버그 해결.
- **업데이트 그룹화 최적화**: 복사-붙여넣기(`bulkUpdateData`) 시 셀 단위가 아닌 행 단위로 업데이트를 묶어 전송함으로써 서버 부하 및 실시간 신호 폭주 방지.
- **부상(Floating) 순서 보존**: WebSocket 리스트를 역순(`reversed`)으로 순회 처리하여, 여러 행 수정 시 사용자의 의도대로 상하 순서가 유지된 채 최상단으로 올라오도록 개선.

## 3. 결과 및 기대 효과
- **사용자 경험(UX) 개선**: 실시간 업데이트 중에도 시간 정보가 사라지지 않아 데이터 신뢰도 확보.
- **데이터 무결성**: 복차-붙여넣기 등 대량 작업 시에도 행 복제나 순서 뒤집힘 없이 정합성이 유지됨.
