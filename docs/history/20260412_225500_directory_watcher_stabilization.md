# 프로젝트 이력: 실시간 디렉토리 워처(Directory Watcher) 결함 해결 및 안정화 (2026-04-12)

## 1. 개요
실시간 로그 감지 및 자동 인제션을 담당하는 `directory_watcher.py`의 비정상 동작을 해결하고, 윈도우 환경에서의 작동 신뢰성을 확보함.

## 2. 상세 작업 내용

### 2.1 설정 파일 탐색 로직 유연화
- **문제 현상**: `sensor_metrics` 워크스페이스의 설정 파일명이 `sensor_config.json`인 경우 워처가 이를 인식하지 못하고 감시 대상에서 제외함.
- **해결 방안**: 고정된 파일명(`config.json`) 대신 `config/` 디렉토리 내의 첫 번째 JSON 파일을 자동으로 찾아 바인딩하도록 로직을 개선함.

### 2.2 런타임 에러 (`NameError`) 수정
- **문제 현상**: 설정 파일 탐색 로직 수정 과정에서 `archives_path` 변수 정의가 누락되어 런타임 시 프로세스가 종료됨.
- **해결 방안**: 변수 정의 누락 구문을 재삽입하여 아카이빙 기능의 무결성을 복구함.

### 2.3 윈도우 중복 이벤트 방어 및 안정화
- **문제 현상**: 윈도우 환경에서 파일 이동/수정 시 다중 이벤트가 발생하여 동일 파일에 대해 인제션이 중복 트리거되고, 이 과정에서 "파일을 찾을 수 없음" 에러가 반복 발생함.
- **해결 방안**:
  - 과도하게 민감한 `on_modified` 이벤트를 제거함.
  - `processing_files` 셋(Set)을 도입하여 현재 처리 중인 파일의 중복 진입을 원천 차단함.
  - 디바운스(Debounce) 이후 파일 존재 여부를 재검증하는 로직을 추가함.

## 3. 관련 파일 변경 사항
- `server/parsers/directory_watcher.py`: 설정 파일 자동 탐색, 중복 방어 로직, 상세 디버그 로깅 추가.

## 4. 최종 결과
- **정상 작동 확인**: `sensor_metrics/raws`에 파일 추가 시 즉시 감지 -> 인제션 수행 -> WebSocket 브로드캐스트 -> `archives/` 이동 처리가 단일 트랜잭션으로 깔끔하게 수행됨.
- **가시성**: 터미널 로그를 통해 감시 중인 워크스페이스와 설정 파일 현황을 명확히 확인할 수 있음.
