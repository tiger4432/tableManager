# 🏁 2026-04-26 20:33:09 - 실시간 업데이트 시스템 컬럼 보강 및 시간 형식 표준화

실시간 데이터 업데이트 과정에서 시스템 컬럼(`created_at`, `updated_at`)이 누락되어 UI 상에서 정보가 사라지던 문제를 해결하고, 시간 형식을 일관성 있게 조정하였습니다.

## 🛠️ 주요 변경 사항

### 1. 실시간 업데이트 페이로드 보강 및 시간 형식 표준화 (Server)
- **시스템 컬럼 추가**: 기존에 `batch_row_upsert` 이벤트 페이로드(`items`)에서 누락되었던 `created_at` 및 `updated_at` 필드를 명시적으로 추가하였습니다. 이를 통해 소스 삭제/수정 등 실시간 이벤트 발생 시에도 시간 정보가 온전히 유지됩니다.
- **시간 형식 표준화**: 브로드캐스트되는 시간 형식을 기존 ISO 형식(`2026-04-26T20:30:59.123`)에서 서버 표준 형식(`2026-04-26 20:30:59`)으로 통일하였습니다 (`to_local_str()` 적용).
- **대상 엔드포인트**: `apply_batch_updates_endpoint`, `delete_cell_source`, `set_cell_priority` (모든 주요 변경 브로드캐스트 경로)

### 2. 클라이언트 업데이트 안정성 보강 (Client)
- **데이터 병합 로직 개선**: WebSocket으로 수신된 `item` 전체를 기존 행에 병합(`update`)하도록 하여 시스템 컬럼 유실을 방지합니다.
- **방어 코드 추가**: 아직 데이터가 로딩되지 않은 행(None 객체)에 대해 업데이트가 유입될 경우 발생할 수 있는 런타임 에러를 방지하였습니다 (`row is None` 체크).
