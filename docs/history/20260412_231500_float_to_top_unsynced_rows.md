# 프로젝트 이력: 비가시 데이터 실시간 부상(Float-to-top) 로직 구현 (2026-04-12)

## 1. 개요
지연 로딩(Lazy Loading) 방식의 테이블에서 현재 로딩되지 않은 하단부 데이터가 수정될 경우, 사용자가 이를 실시간으로 인지하지 못하는 문제를 해결함.

## 2. 상세 작업 내용

### 2.1 단일 행 페칭 워커 (`ApiSingleRowFetchWorker`) 신설
- 특정 `row_id`를 기반으로 서버에서 해당 행의 전체 데이터를 가져오는 전용 워커를 구현함 (`GET /tables/{table_name}/{row_id}`).

### 2.2 비가시 데이터 감지 및 플로팅(Floating) 메커니즘
- **로직**: WebSocket 브로드캐스트 수신 시, 해당 `row_id`가 현재 클라이언트 버퍼(`_data`)에 없는 경우(로딩 안 됨)를 체크함.
- **동작**: 버퍼에 없더라도 수정 이벤트라면 무시하지 않고 `ApiSingleRowFetchWorker`를 실행하여 서버에서 데이터를 가져옴.
- **반영**: 데이터 수신 완료 후 `beginInsertRows`를 통해 테이블의 최상단(0번 행)에 해당 데이터를 강제 삽입함.

### 2.3 최적화 (Duplicate Fetch Prevention)
- `_fetching_row_ids` 세트를 도입하여, 동일 행에 대해 짧은 시간 내 여러 업데이트가 올 경우 중복 페칭이 일어나는 것을 방지함.

## 3. 최종 결과
- 테이블 하단에 있어 로딩되지 않은 데이터가 수정되더라도, 수정 즉시 해당 데이터가 화면 최상단으로 '부상(Float)'하여 나타나는 것을 확인함.
- 사용자는 전체 데이터 규모와 상관없이 모든 실시간 변경 사항을 최상단에서 모니터링할 수 있음.
