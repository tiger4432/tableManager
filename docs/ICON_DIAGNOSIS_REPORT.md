# 🕵️ AssyManager: 아이콘 미출력 원인 분석 보고서

본 보고서는 AssyManager Enterprise Edition 실행 시 아이콘이 출력되지 않는 현상에 대한 기술적 분석 결과를 담고 있습니다.

---

## 1. 현상 분석 (Summary of Issues)

메인 윈도우 좌측 상단 및 윈도우 작업 표시줄(Taskbar)에 지정된 `app_icon.png`가 나타나지 않고 기본 파이썬 아이콘(또는 빈 아이콘)이 지속적으로 노출되는 현상이 보고되었습니다.

---

## 2. 주요 기술적 원인 (Root Cause Analysis)

심층 조사 결과, 다음과 같은 세 가지 주요 원인이 복합적으로 작용했을 가능성이 큽니다.

### 🔍 원인 A: Qt Image Format 플러그인 누락 (가장 유력)
- **증상**: `QIcon` 객체가 생성은 되나, 실제 이미지 파일을 디코딩하지 못함.
- **분석**: PySide6(Qt)는 PNG 파일을 로드할 때 `imageformats/qpng.dll` 플러그인을 사용합니다. 현재 Conda 환경에서 이 플러그인이 로드되는 경로가 시스템 PATH에 포함되어 있지 않거나, DLL 격리 로직에 의해 차단되었을 때 이미지가 "공백(Null)"으로 처리되어 아이콘이 나오지 않습니다.

### 🔍 원인 B: 윈도우 셸의 아이콘 캐시 (Windows Shell Caching)
- **증상**: 코드가 정상이고 파일도 존재하지만 아이콘이 바뀌지 않음.
- **분석**: 윈도우는 특정 `AppUserModelID`를 가진 프로세스의 아이콘을 캐시합니다. 개발 과정에서 이미 다른 아이콘(또는 기본 아이콘)으로 한 번 인식되었다면, 캐시가 갱신되기 전까지는 새로운 아이콘을 시각적으로 투영하지 않는 경우가 발생합니다.

### 🔍 원인 C: 이미지 메타데이터 및 포맷 호환성
- **증상**: 일부 뷰어에서는 보이지만 Qt에서만 안 보임.
- **분석**: 생성된 PNG 파일이 고해상도(1024x1024 이상)이거나, AI 생성 과정에서 특수한 컬러 프로파일(ICC Profile) 또는 16-bit 채널을 포함할 경우 Qt의 기본 디코더가 이를 무시하거나 로드에 실패할 수 있습니다.

---

## 3. 해결을 위한 제안 (Proposed Solutions)

### ✅ 조치 1: 윈도우 표준인 .ico 포맷으로 전환
- PNG 대신 다양한 해상도(16/32/48/256px)가 하나의 파일에 패키징된 **`.ico` (Icon Collection)** 파일을 생성하여 적용합니다. 이는 윈도우 환경에서 가장 안정적인 방식입니다.

### ✅ 조치 2: 플러그인 로드 경로 명시적 등록
- `main.py` 상단에서 `QApplication` 초기화 전, Qt 플러그인 디렉토리를 명시적으로 추가하여 디코딩 라이브러리를 강제 로드합니다.
  ```python
  QCoreApplication.addLibraryPath(os.path.join(pyside_dir, "Qt", "plugins"))
  ```

### ✅ 조치 3: 아이콘 로드 검증 로직 추가
- 아이콘 로딩 성공 여부를 로그로 남기거나, 실패 시 기본 아이콘으로 Fallback 하는 방어 코드를 작성합니다.
